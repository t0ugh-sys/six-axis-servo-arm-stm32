# 串口命令协议（UART）

本仓库提供一个非常轻量的“行命令”解析器：
- `firmware/Inc/arm_cmd.h`
- `firmware/Src/arm_cmd.c`

你可以用它在 PC/手机端通过串口控制机械臂（不依赖调试器改变量）。

## 基本规则

- 发送方式：**ASCII 文本**
- 一条命令是一行：必须以 `CRLF` 或 `LF` 结束（也就是回车/换行）
- Token 之间用空格分隔
- 大小写不敏感（`J`/`j`，`U`/`u`，`T`/`t` 都可以）

## 命令含义（J / U / T）

### U：直接脉宽（us）

格式：
- `U<ch>=<us>`

含义：
- 直接设置 PCA9685 的某个通道输出脉宽（单位微秒 `us`）
- 常用于找中位、安装舵盘、排查问题（绕过角度映射）

示例：
- `U0=1500`
- `U0=1500 U1=1600`

### J：关节角度（deg）

格式：
- `J<idx>=<deg>`

含义：
- 设置某个关节的目标角度（单位度 `°`），由 `arm.c` 映射为舵机脉宽

示例：
- `J0=60`
- `J0=90 J1=120 J2=90`

### T：动作时间（ms）

格式：
- `T=<ms>`

含义：
- 插补运动用时（单位毫秒），越大越慢越平滑

示例：
- `J0=60 T=800`
- `J0=90 J1=90 T=1000`

## 常见踩坑

### 1) 没有 `OK/ERR`、命令粘在一起

原因：你没发送真正的换行（`CR`/`LF`），解析器不会执行。

解决：在串口工具里勾选“发送行尾”：
- `CRLF` 或 `LF`（二选一）
- 不要手打 `\\r\\n` 字符串

### 2) PuTTY 方向键/退格导致 `ERR`

方向键、退格会发送控制序列（例如 `ESC [ A`），混入命令行导致解析失败。

建议：
- 直接复制粘贴整行命令再回车
- 或使用仓库提供的 `examples/cube_main_uart_arm_debug.c`（已过滤 ANSI ESC 序列，并处理 DEL/Backspace）
